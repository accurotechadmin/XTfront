# Parity Retrospective Document Set

This folder contains the requested retrospective documents from the frontend/backend parity execution cycle:

1. `01_backend_team_learnings_and_changes.md` — what backend had to learn, produce, and change.
2. `02_frontend_team_spec_and_match_requirements.md` — what frontend had to specify and match exactly.
3. `03_joint_wisdom_and_reusable_playbook.md` — reusable wisdom, anti-patterns, and starter guidance.

These documents are designed to support checkpoint handoff, onboarding, and future parity programs.
